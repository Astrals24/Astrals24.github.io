    顺序表

    a.删除线性表中所有值为x的数，时间复杂度O(n),空间复杂度O(1) 王道P22

    解法一：用K记录L中不等于x的个数（即最后需要保存的元素个数），边扫描L边统计K，并把不等于x的元素赋值到k位置上，最后设置长度为K。

    解法二：用K记录L中等于x的个数，边扫描L边统计K，将不等于x的元素前移k个位置（即赋值到i-k的位置上 相当于越过了等于x的数），最后设置长度为L.length-K。

    b.从==有序表==中删除所有其重复的元素，使表中所有元素值均不相同

    解法：类似直接插入排序的思想，初始将第一个元素视为非重复有序表（用i记录该表的大小），依次判断后面的元素（用j遍历）是否与前面i指向的元素重复，若不重复则让++i 并把元素赋值到i上（即有序表的最后）。

    c.将两个有序表合并为一个新的有序表

    解法：按顺序取2个表的较小的结点存入新的有序表中，最后剩下的那个表所有元素依次再插入新表。

    d.已知一维数组A[m+n]中依次存放两个线性表(a1,a2,a3,….am) 和 (b1,b2,b3,….bn)，将数组中两个顺序表位置互换，即将(b1,b2,b3….bn)放在(a1,a2,a3,….am)前面

    解法：把A[m+n]先原地逆置 (bn,…..,b1,am,….,a1) 再对前n个元素与后m个元素分别逆置即能实现位置互换。

    e.将n个整数保存到一维数组R中，将R中保存的序列==循环左移==p个位置，即将R中的数据由(X0,X1…,Xn-1) 转为(Xp,Xp+1…Xn-1,X0,X1)

    解法：将循环左移问题转化为数组ab变ba（a代表前P个元素，b代表后n-p个元素） 先分别对a，b转置得$a^-b^-$ 再对其整体转置$ (a^-b^-)-$

    得ba 即为所求

    f.给定一个含n个(n>=1)整数的数组A，设计一个在时间上尽可能高效的算法，找出数组中未出现的最小正整数 例如 {-5,3,2,3}最小正整数是1

    解法：假定数组A中的数大小介于1到n中（若不在1~n中 则最小正整数为1）利用数组Bn, B[0] 对应正整数 1，B[n-1] 对应正整数n，当数字不在1到n中时不做任何操作，否则把对应该数字的数组下标的数组值+1 最后遍历Bn 遇到第一个数组值为0的 数组下标即为未出现的最小正整数（值为0 表示该数从未出现过）。

    链表

    a.单链表

    带头结点（结点内不存储信息），不带头结点

    缺点:只有后继指针，没有前驱指针，只能通过重新遍历方式找到其前驱结点，效率低

    b.双链表

    结点有两个指针prior和next，分别指向前驱和后继

    变化：能很方便的访问结点的前驱和后继 因此 插入和删除操作的时间复杂度都为O(1)

    c.循环单链表

    判空条件：由于尾结点的next指向L 所以表中没有指针域为NULL结点 当头指针**==指向头结点==**时链表为NULL

    变化：循环链表可以从任意一个结点开始遍历整个链表，而单链表操作喜欢在头或尾进行，所以**==不设头指针只设置尾指针会使操作效率更高==** 原因是若设置头指针 找表尾需要O(n) 而尾指针对表头和表尾都只需要O(1)

    d.循环双链表

    判空条件：头结点的prior与next都指向L

    e.静态链表

    特性

    • 借助数组来描述线性表的链式存储结构
    • 需要预先分配连续的内存空间
    • 插入，删除操作只需要移动指针，不需要移动元素

    选择题技巧

    • 见到在链表末尾插入删除，在头插入删除，需要末尾前驱的，循环双链表最快 其次单循环链表

    • XXX最后一个元素：需尾结点的前驱 -> 循环

    • XXX第一个元素： 需头结点 -> 双链表尾,头或循环尾

    如果节点有右孩子，则指向右孩子 无右孩子则指向先序后继 （若有左无右，则右指针仍然指向左 孩子）

    图

    基本性质

    完全图：对于无向图：任意两个顶点之间都存在变 最大边 n（n-1）/2

    ​ 对于有向图：任意两顶点之间都存在方向相反的两条弧 最大边n（n-1）

    连通，连通图和连通分量：无向图中任意两个顶点都连通，则为连通图，否则，图中的极大连通子图为连通分量

    若图中有n个顶点，并且边数小于n-1，此图必是非连通图。

    强连通图和强连通分量：有向图中，每一对顶点都有路径，则称为强连通图，否则，图中的极大连通子图称为强连通分量

    边数至少 n （最少边构成环）

    极小连通子图：添加一条边就会有环

    环： n个顶点n条边，去任意一条便是一颗生成树，有n种情况的生成树。

    环：第一个顶点和最后一个顶点相同的路径称为回路或环，有n个顶点，大于n-1条边，一定有环

    顶点的度，入度，出度：度为以该顶点为一个端点的边的数目， 度之和为入度+出度

    具有n个顶点，e条边的无向图，全部顶点度之和为2e

    具有n个顶点，e条边的有向图，全部顶点度之和为e

    生成树：一个连通图的生产树是一个极小连通子图，显然它是无环的。

    汇总：

    1.若图中有n个顶点，并且边数小于n-1，此图必是非连通图

    2.有n个顶点，大于n-1条边，一定有环

    3.具有n个顶点，e条边的无向图，全部顶点度之和为2e

    具有n个顶点，e条边的有向图，全部顶点度之和为e

    4.强连通图，边数至少 n （最少边构成环）

    5.n个顶点图是个环，有n棵生成树

    6.n个顶点，e条边的无向图是个森林，森林必有n-e棵树

    名称 时间复杂度 空间复杂度 稳定性 备注
    直接插入排序 O(N)~O(N^2) O(1) 稳定 表越有序，越快
    折半插入排序 O(N^2) O(1) 稳定 相对于插入排序减少了比较元素的次数
    希尔排序 O(N^(1.3)) ~ O(N^2) O(1) 不稳定 优化了直接插入排序

    卡兰特数详解

    卡特兰数简介

    卡特兰数是组合数学中的一种著名数列，通常用如下通项式表示(为了不与组合数C CC冲突，本文用f ff表示卡特兰数)：
    f ( n ) = C 2 n n n + 1 f(n)=\frac{C_{2n}^{n}}{n+1}f(n)=n+1C2n**n​​
    当然，卡特兰数也是有递推式的：
    f ( n ) = ∑ i = 0 n − 1 f ( i ) × f ( n − i − 1 ) f(n)=\sum_{i=0}^{n-1}f(i)\times f(n-i-1)f(n)=i=0∑n−1​f(i)×f(n−i−1)
    但在实际应用中，最常用的却是第一个通项式的变形：
    f ( n ) = C 2 n n − C 2 n n − 1 f(n)=C_{2n}^{n}-C_{2n}^{n-1}f(n)=C2n**n​−C2n**n−1​

    卡特兰数的应用

    基本模型

    有一个长度为2 n 2n2n的01序列，其中1,0各n nn个，要求对于任意的整数k ∈ [ 1 , 2 n ] k \in [1,2n]k∈[1,2n]，数列的前k kk个数中，1的个数不少于0。

    解决方法

    当然你可以选择用DP秒切。

    不知当初哪位神仙想出了这种神奇的构造方法。

    我们把0,1操作扔到一个坐标系中。1看成向右上方走一步，0看成向右下角走一步，那么最后构造完后一定走到了( 2 n , 0 ) (2n,0)(2n,0)

    如下图：

    那么总的路径数量就是在2 n 2n2n步中选择n nn步为1，方案数为C 2 n nC_{2n}^{n}C2n**n。

    接着来考虑题目中的限制条件：对于任意前缀，1的个数不少于0。

    那么转化到坐标系中，也就是说走的路径不应该穿过x xx轴，即直线y = 0 y=0y=0，也就是不接触y = − 1 y=-1y=−1。

    于是我们把与y = − 1 y=-1y=−1的接触点的右边整条路径以y = − 1 y=-1y=−1为对称轴翻折，于是终点变为了( 2 n , − 2 ) (2n,-2)(2n,−2)。如下图：

    那么此时，从( 0 , 0 ) (0,0)(0,0)到( 2 n , − 2 ) (2n,-2)(2n,−2)的路径必定至少穿过一次y = − 1 y=-1y=−1，不满足条件，那么此时的路径数量即为总路径数中不符合题意的路径数，那么我们用总路径数减去不符合的路径数，就是最终的答案。

    而此时的路径数量也很简单，由于反转后终点向下移了两位，也就意味着n − 1 n-1n−1步是1，n + 1 n+1n+1步是0，总方案为C 2 n n − 1 C_{2n}^{n-1}C2n**n−1，那么最终的答案就是C 2 n n − C 2 n n − 1C_{2n}^{n}-C_{2n}^{n-1}C2n**n−C2n**n−1。

    诶，这不就是卡特兰数吗。

    1. 洛谷P2532 树屋阶梯

    这道题我一看两军阵前……，跟卡特兰数好像没有直接关系，但是手画几个样例后，发现答案都是卡特兰数，那么基本上就可以大胆猜想了。

    但实际上，我们可以从d p dpd**p的角度考虑这道题。对于任意大小为n nn的阶梯，我们都可以现在左下角放一个大块，然后在上方构造一个大小为i ii的阶梯，右下方构造大小为n − i − 1 n-i-1n−i−1的阶梯，那么转移方程就十分显然：
    f ( n ) = ∑ i = 0 n − 1 f ( i ) ∗ f ( n − i − 1 ) f(n)=\sum_{i=0}^{n-1}f(i)f(n-i-1)f(n)=i=0∑n−1​f*(i)∗f(n−i−1)
    显然就是卡特兰数，直接一波通项式+高精度搞定。

    1. 洛谷P3200 有趣的数列

    首先有一个结论，对于每个偶数位，上面放的数必须不小于当前位的下标。（意会一下）

    转化一下题意：把1~2n这些数依次放入序列，每次可以选择最后的奇数位或偶数位。

    然后再观察后发现一个显然的性质：对于任意时刻，数列中的奇数位数量不能少于偶数位数量。否则偶数位上放的数不可能大于等于下标。

    于是，这就是一个卡特兰数模板，通项式秒切。具体对于组合数的处理，可以参看各种题解。

    6、已知一棵完全二叉树的第6层（设根为第1层）有8个叶结点,则完全二叉树的结点个数最多是( )。

    A）39 B）52 C）110 D）111

    最少 39

    最多 111

    需要认真考虑的一道题。第六层有八个叶节点，其实有两种情况：

    1、第六层只有六个节点，且为叶节点（常想到的）——节点最少情况；

    2、第六层满节点，但是其中的八个节点没有孩子节点，因此其也是叶节点（不易想到）——节点最多情况；

    本题正是考察第二种情况。

    首先，前六层满节点，那么节点个数为：26-1=63

    其次，第六层有26-1 =32个节点，其中有8个没有孩子节点，说明第七层有（32-8）*2=48个节点。

    因此，最多情况下共有：63+48=111个节点

    1.出栈顺序的合法性检验

    一、出栈序列判断
    问题：按1、2、3、4、5进栈，出栈是否能得到1、2、3、4、5？是否能得到3、4、5、1、2？
    答案：可以得到1、2、3、4、5，只要1进栈，1出栈，2进栈，2出栈以此类推即可得到1、2、3、4、5；但是不能得到3、4、5、1、2（为什么？）。
    二、算法思想
    如果使用暴力破解的方法，n个数的进栈序列，可以有C(2n,n)/(n+1)个（卡特兰（Catalan）数），然后判断出栈序列是否属于其中，但是复杂度过大。
    利用编译原理中判断表达式是否合法的思路，再利用一个辅助栈即可解决问题。首先用两个指针分别指向进栈序列和出栈序列的初始位置，然后遍历进栈序列，先把每个进栈序列的元素压入辅助栈中，接着遍历栈中元素，条件是出栈序列的指针还未溢出而且辅助栈栈顶元素等于出栈序列的指针所指向的值，辅助栈执行出栈操作并且出栈序列的指针后移（这里说的指针有点合适），最后判断辅助栈是否为空，如果为空，出栈序列合法，如果不为空，不合法。下面用表格来详细解释3、4、5、1、2为什么不合法。

    进栈序列遍历结束，辅助栈中还有元素，说明该出栈序列不合法。（1在2之前进栈，由于栈先进后出的规则，不可能在2之前出栈）
    【注】其中粗体数字表示当前指针指向的位置
