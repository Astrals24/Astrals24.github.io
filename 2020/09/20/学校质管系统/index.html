教师教学质量管理系统

项目背景

2020年3月，我所在实验室应学校质管处委托承接了某教师质量管理系统的开发工作，该系统受学校全面推进信息化建设政策指导，意在为学校质管处工作人员针对教师质量管理提供全方位的软件支撑。系统于2020年3月开始建设，2020年9月顺利通过验收正式上线运行，历时半年，本人负责整个系统开发与设计，开发语言为C#，CS架构，使用winform开发。由于项目性质，暂时无法开源。

项目介绍

项目主要为了方便质管处工作人员对学校各教师的教学质量评估工作进行统计与分析，大致分为三个主要模块

  • 质量管理模块
    • 手动录入教学质量信息
    • EXCEL批量导入教学质量信息
    • 统计与导出教学质量信息等
  • 人员管理模块
    • 教师基本信息管理
    • 账号基本信息管理
    • 工作人员质管权限信息管理等
  • 评教项管理模块
    • 设置质管评教项
    • 设置学院名
    • 设置当前学期等

系统还区分两类角色管理员与超级管理员，管理员只能使用质量管理这个模块，超级管理员则没有使用限制。

项目主要使用技术

1、MySQL 双机热备

原理一览

​ 双机热备首先要实现主从复制，当实现主从复制后，反过来用从机当master，主机当slave 再做一遍主从复制即可实现双机热备。

​ 主从复制成功与否的标志就是右图箭头所指io_Run与SQL_Run状态是否都为Yes。

用途

​ 由于项目性质，数据极其重要，且学校服务器时常不稳定，在关键时候用户可能很难容忍长时间的服务中断，为了保障数据的安全性与服务的稳定性项目通过双机热备服务，来避免长时间的服务中断，保证系统能长期、可靠运行。

2、MyCat 读写分离

原理一览

​ 读写分离就是把对数据库的读和写操作分开以对应不同的数据库服务器。主数据库提供写操作，从数据库提供读操作，这样能有效地减轻单台数据库的压力，当然前提是需要主从复制。

​

用途

​ 主要用于保障部分热点功能的高性能与高可用。

3、Docker

原理一览

用途

​ 主要用于快速部署数据库避免繁琐的MySQL安装，并可通过开启多个端口创建多个MySQL容器以测试MySQL主从功能是否正确开启等。

4、Entity Framework

原理一览

​ Entity Framework 简称EF是微软用来替代ADO.NET的一套 ORM 框架.可以在 SQL Server、MySQL、Oracle、等数据库上使用。可以将数据作为业务对象和实体进行操作，使用LINQ进行查询，使用C#进行操作和检索。

​ EF原理图

用途

​ ORM框架不多说，让我少写了很多CRUD重复代码，减少了很多工作量，官方语言是让程序员更多的关注业务实现，缺点是面对复杂SQL不太灵活，所以本项目中采用的是EF与ADO.NET混用形式。（可能是我对EF不太会用的缘故）

项目所遇问题及解决方案

1、工作人员评教权限功能表设计冗余

问题描述：由于管理员需要对工作人员的不同学院，不同评价类别，不同学期评教的读写权限进行升降，导致了每新注册一个工作人员就要新增学院数X评价类别数X学期数条权限记录，并且由于该记录的唯一性无法再次进行表拆分。貌似符合多值依赖？

方案：还未找到好的解决方案，只能先把学期单独分离出去，只保留学院-评价类别-权限，坏处是无法设置学院-评价类别单独一条记录在不同学期的权限。

2、学校服务器不稳定服务时长中断

问题描述： 学校服务器时常抽风导致无法连接数据库，每次登录不上就会找我排查问题，排查一半发现又登录的上了，最后发现是学校问题非常浪费时间。

方案： 用实验室自己的服务器当从机，使用双机热备技术，学校服务出现问题后自动切换至实验室服务器。（除非都出现故障）

后记

​ 项目从开始设计到最后上线历时半年，虽说历时半年，但其实最初的版本一个星期就完成了，但由于项目前期没有保留正式的需求分析书面材料，只有口头交流和一些简单的简记，导致主体功能完成后交付测试时需求又一改再改！有时候甚至出现否定自己最开始的需求的事情发生，当然也可能是我自己理解有问题。特别是比较重大的功能变化时需要重新设计数据库表，让人欲哭无泪。

​ 这段经历成功让我养成了每次设计完表都会多加几个冗余字段以备不时之需的习惯虽然不知道是好是坏，还有就是让我明白了”需求分析占项目的50%”这个道理，没有明确的需求分析文档，一定不要急着开始！
